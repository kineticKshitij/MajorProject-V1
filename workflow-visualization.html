1

User Creates Crawl Job

User accesses the web interface and creates a new crawl job by selecting:

  • Target platform (Twitter, GitHub, LinkedIn, Reddit)
  • Search keywords or Google dork query
  • Optional: Enable/disable Tor (default: enabled)
2

Django Backend Receives Request

The Django REST API processes the request:

POST /api/crawl/ { "platform": "twitter", "query": "data breach", "use_tor": true }

Backend validates the request and queues it for processing.

3

Tor Service Initialization

If Tor is enabled, the system:

  • Connects to Tor proxy (localhost:9050)
  • Verifies Tor connection is active
  • Gets current exit node IP
  • Configures SOCKS5 proxy for all requests
4

Web Crawling Execution

The appropriate crawler is invoked:

# Automatic Tor usage crawler = TwitterCrawler(use_tor=True) results = crawler.search(query="data breach") # Your real IP: 202.142.121.121 # Tor exit IP: 185.195.71.244 ✅

All requests go through Tor - your real IP is completely hidden!

5

Data Extraction & Processing

The crawler extracts relevant information:

  • URLs, titles, descriptions
  • Social media posts, profiles, metadata
  • GitHub repositories, code snippets
  • Reddit discussions, comments

All data is cleaned, structured, and validated before storage.

6

Database Storage

Results are stored in PostgreSQL database:

CrawlJob (id, status, platform, query, created_at) └─ CrawlResult (url, title, content, extracted_at)

Each result is linked to the original crawl job for tracking.

7

Circuit Rotation (Optional)

For enhanced anonymity, Tor circuits rotate automatically:

  • Default: Every 60 seconds
  • Manual: On-demand via API call
  • Automatic: After X requests (configurable)

Results Delivery

User receives extracted data via:

  • Web dashboard with real-time updates
  • REST API endpoints for programmatic access
  • Export options (JSON, CSV, Excel)

🚀 Deployment Guide

1

Prerequisites

Ensure you have the following installed:

  • Docker Desktop (Windows/Mac) or Docker Engine (Linux)
  • Docker Compose v2.0+
  • Python 3.11+
  • Git
2

Clone Repository

git clone https://github.com/kineticKshitij/MajorProject-V1.git cd MajorProject-V1
3

Configure Environment

# Copy example environment file cp .env.example .env # Edit .env with your settings # Key settings: TOR_ENABLED=true TOR_PROXY_HOST=localhost TOR_PROXY_PORT=9050 DJANGO_SECRET_KEY=your-secret-key-here
4

Start Tor Services

# Start Tor proxy in Docker docker-compose -f docker-compose.tor.yml up -d # Verify containers are running docker ps # Expected output: # ✅ tor-proxy (healthy) # ✅ django-app (healthy)
5

Install Python Dependencies

# Create virtual environment python -m venv env # Activate (Windows PowerShell) .\env\Scripts\Activate.ps1 # Activate (Linux/Mac) source env/bin/activate # Install dependencies pip install -r requirements.txt
6

Database Setup

# Run migrations python manage.py migrate # Create superuser python manage.py createsuperuser # Load initial data (optional) python manage.py loaddata initial_dorks.json
7

Start Django Server

# Development server python manage.py runserver # Access at: # 🌐 Web Interface: http://localhost:8000 # 🔧 Admin Panel: http://localhost:8000/admin
8

Verify Tor Integration

# Quick verification python test_tor_demo.py # Full test suite python test_crawler_automation.py # Expected: All tests pass ✅
